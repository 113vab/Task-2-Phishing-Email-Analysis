# Sample 1: Fake PayPal Verification Email

## Email Details

**Sender:**
[security-update@paypa1.com](mailto:security-update@paypa1.com)

**Subject:**
Urgent: Your Account Has Been Limited

### Summary

The email claims that unusual activity has been detected on the recipient's PayPal account and requests immediate verification through a provided link.

### Phishing Indicators Identified

| Indicator           | Observation                                            |
| ------------------- | ------------------------------------------------------ |
| Email Spoofing      | "paypa1.com" uses the number 1 instead of the letter l |
| Urgent Language     | "Verify immediately"                                   |
| Threatening Content | Account suspension warning                             |
| Suspicious URL      | Non-official PayPal website                            |
| Generic Greeting    | "Dear Customer"                                        |
| Social Engineering  | Fear and urgency tactics                               |

### Security Risks

* Credential theft
* Financial fraud
* Identity theft

### Conclusion

The email demonstrates several phishing characteristics and attempts to steal user credentials through a fake verification page.

# Sample 2: Fake SBI Banking Alert

## Email Details

**Sender:**
[support@sbi-security-alert.com](mailto:support@sbi-security-alert.com)

**Subject:**
Urgent: Your SBI Account Has Been Locked

### Summary

The email informs the recipient that their bank account has been locked and requests immediate verification to restore access.

### Phishing Indicators Identified

| Indicator          | Observation                      |
| ------------------ | -------------------------------- |
| Fake Sender Domain | Not an official SBI domain       |
| Urgency            | Immediate verification requested |
| Threat             | Permanent account suspension     |
| Suspicious URL     | Fake banking website             |
| Generic Greeting   | "Dear Customer"                  |
| Social Engineering | Fear-based manipulation          |

### Security Risks

* Banking credential theft
* Unauthorized transactions
* Financial loss

### Conclusion

The email attempts to create panic and direct users to a fraudulent website designed to capture banking credentials.

# Sample 3: Microsoft 365 Password Expiry Scam

## Email Details

**Sender:**
[admin@microsoft365-support.net](mailto:admin@microsoft365-support.net)

**Subject:**
Password Expiration Notice

### Summary

The email claims that the user's Microsoft 365 password is about to expire and requests an immediate password update through a provided link.

### Phishing Indicators Identified

| Indicator             | Observation                           |
| --------------------- | ------------------------------------- |
| Fake Microsoft Domain | Not an official Microsoft domain      |
| Urgent Request        | Password update required immediately  |
| Threat                | Account deactivation warning          |
| Suspicious URL        | Fake password reset portal            |
| Generic Greeting      | "Dear User"                           |
| Credential Harvesting | Attempts to collect login credentials |

### Security Risks

* Account compromise
* Unauthorized access to emails
* Data theft
* Cloud account takeover

### Conclusion

The email is a phishing attempt designed to steal Microsoft account credentials using urgency and fake account maintenance notifications.

# Common Phishing Characteristics Observed

1. Fake sender email addresses.
2. Generic greetings instead of personalized names.
3. Suspicious links that do not match official domains.
4. Urgent language encouraging immediate action.
5. Threats of account suspension or deactivation.
6. Requests for sensitive information or credentials.
7. Use of social engineering tactics such as fear and urgency.

---

# Security Recommendations

1. Verify sender email addresses carefully.
2. Never click links from unsolicited emails.
3. Hover over links to inspect actual URLs.
4. Access services directly through official websites.
5. Enable Multi-Factor Authentication (MFA).
6. Report suspicious emails to the appropriate organization.
7. Keep systems and browsers updated.
8. Educate users about phishing awareness.
